# Внутренние подписанные PDF

Эта инструкция описывает, как подготовить внутренние PDF-документы для цифровой подписи и где получить результат.

## Куда положить файлы

Внутренние документы нужно размещать в папке:

```text
/data/www-provisioned/Web/umo/internal/
```

Внутри папки `internal` нужно создать отдельную папку для каждого набора документов. Папки можно вкладывать друг в друга, если так удобнее для организации документов. Названия папок могут быть любыми понятными названиями.

Пример структуры:

```text
umo/
  internal/
    dogovory-2025/
      index.csv
      dogovor-1.pdf
      dogovor-2.pdf
    prikazy-avgust/
      index.csv
      prikaz-1.pdf
    outer/
      inner/
        index.csv
        asset.pdf
```

PDF-файлы должны лежать прямо в той же папке, что и соответствующий `index.csv`. Если внутри есть дополнительные вложенные папки, они будут обработаны отдельно только при наличии собственного `index.csv`.

## Как подготовить index.csv

В каждой папке с PDF-файлами должен быть файл `index.csv`. Если его нет, эта папка будет пропущена.

Файл `index.csv` должен содержать заголовок и ровно одну строку с данными:

```csv
дата_подписи;имя_подписавшего;должность_подписавшего
26.08.2025;Александр Михайлович Копировский;Ректор
```

Разделитель колонок: точка с запятой `;`.

Дата указывается в формате `дд.мм.гггг`.

Данные из `index.csv` применяются ко всем PDF-файлам в этой же подпапке.

## Как запустить генерацию

1. Откройте модуль импорта УМО в административной части сайта.
2. Если нужно сначала проверить, какие файлы будут найдены, нажмите кнопку `Проверить внутренние PDF без генерации`.
3. Для запуска генерации нажмите кнопку `Генерировать внутренние подписанные PDF`.
4. Генерация запустится в фоновом режиме.

Внизу страницы модуля показывается путь к рабочей папке, путь к логу и последние строки лога. После запуска генерации можно нажимать `Обновить лог`, чтобы увидеть текущий результат.

Лог записывается в файл:

```text
/data/www-provisioned/Web/umo/internal/generation.log
```

## Что появится в результате

В каждой обработанной папке появится подпапка `signed`.

Пример:

```text
umo/
  internal/
    dogovory-2025/
      index.csv
      dogovor-1.pdf
      dogovor-2.pdf
      signed/
        dogovor-1.pdf
        dogovor-2.pdf
    outer/
      inner/
        index.csv
        asset.pdf
        signed/
          asset.pdf
```

В папке `signed` будут лежать подписанные версии PDF-файлов с теми же именами.

Если подписанный файл уже существует, при обычном запуске кнопкой он не перезаписывается.

## Как проверить подпись

Откройте реестр цифровых подписей на сайте и введите ключ документа, указанный в подписанном PDF.

Если документ найден, реестр покажет:

- имя файла;
- ключ документа;
- ФИО подписавшего;
- должность подписавшего;
- дату подписания;
- ссылку для скачивания подписанного PDF.

Внутренние документы не добавляются на публичные страницы УМО. Они становятся доступны через реестр подписи только при проверке по ключу.

## Что делать, если файл не появился

Проверьте:

- есть ли записи в логе на странице модуля;
- папка находится внутри `umo/internal`;
- в папке есть `index.csv`;
- в `index.csv` есть заголовок и одна строка данных;
- дата указана в формате `дд.мм.гггг`;
- PDF-файлы лежат прямо в той же папке, что и `index.csv`;
- подписанный файл с таким именем еще не лежит в папке `signed`.
